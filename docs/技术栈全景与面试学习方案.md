## 三、核心技术深度解析（面试级）

### 3.1 后端：Spring Security + Session 认证

**项目代码位置：**
- `SecurityConfig.java` — 安全配置
- `LoginSuccessHandler.java` / `LoginFailureHandler.java` — 登录处理
- `ProjectAccessService.java` — 细粒度权限控制

**面试会问：**
1. **Session 认证流程是怎样的？**
   - 用户 POST /api/auth/login → Spring Security formLogin 拦截
   - 验证成功 → 服务器创建 HttpSession → 返回 JSESSIONID Cookie
   - 后续请求自动携带 Cookie → 服务器从 Session 中取出用户信息

2. **为什么不用 JWT？**
   - 院内系统不需要无状态/分布式部署
   - Session 天然支持服务端主动踢人（停用用户立即失效）
   - JWT 需要额外处理 Token 刷新、黑名单等

3. **CSRF 为什么关掉了？**
   - `csrf.disable()` — 因为是纯 API 服务，前端通过 Cookie+Ajax 通信
   - 如果是服务端渲染的项目则必须开 CSRF

4. **BCrypt 的原理？**
   - 每次加密生成随机盐值，相同密码两次加密结果不同
   - `passwordEncoder.matches(rawPassword, encodedPassword)` 比对
   - 故意设计成慢算法（~100ms），防止暴力破解

**项目特色代码：**
```java
// Session 登录 — 传统但可靠
.formLogin(form -> form
    .loginProcessingUrl("/api/auth/login")
    .successHandler(loginSuccessHandler)  // 返回 JSON
    .failureHandler(loginFailureHandler)  // 返回 401 + 错误消息
)
```

### 3.2 后端：MyBatis-Plus 核心用法

**项目代码位置：**
- 全部 `mapper/` 文件（BaseMapper）
- 全部 `service/impl/` 文件（ServiceImpl）
- `SysProjectStageServiceImpl.java` — 复杂查询示例

**面试会问：**
1. **MyBatis-Plus 和 MyBatis 的区别？**
   - MP 内置 CRUD，不需要写 XML
   - LambdaQueryWrapper 类型安全，避免字符串拼 SQL
   - 分页插件 `IPage<T>` 自动拦截

2. **LambdaQueryWrapper 怎么防止 SQL 注入？**
   - 不是字符串拼接，是 `eq(SysProject::getName, name)` 这种 lambda 引用
   - 底层用 `PreparedStatement` 参数化查询

3. **N+1 查询问题怎么发现和解决？**
   - 看代码：循环里调了 `xxxMapper.selectById()`
   - 解决：用 `selectBatchIds()` 批量查，再 Java 内存 join

**项目代码示例：**
```java
// LambdaQueryWrapper — 类型安全的查询条件
stageMapper.selectCount(new LambdaQueryWrapper<SysProjectStage>()
    .eq(SysProjectStage::getProjectId, id)
    .ne(SysProjectStage::getStatus, "completed"));

// 分页查询
IPage<SysProject> result = projectService.pageWithMembers(page, size, keyword, status, userId, role);
```

### 3.3 后端：Spring 事务管理

**项目代码位置：**
- `ReportController.java` — `@Transactional` 填报提交
- `OaProjectRowImporter.java` — `@Transactional` 行级导入

**面试会问：**
1. **`@Transactional` 的原理？**
   - Spring AOP 代理：方法前 `begin()`，成功则 `commit()`，抛异常则 `rollback()`
   - 默认只回滚 RuntimeException 和 Error

2. **事务传播行为？**
   - `REQUIRED`（默认）：有事务就加入，没有就新建
   - `REQUIRES_NEW`：总是新建独立事务（本项目 OA 导入使用）
   - `NESTED`：嵌套事务，子事务回滚不影响父事务

3. **为什么填报要加事务？**
   - 一次填报可能同时创建：报告 + 偏差 + 支持事项
   - 任何一步失败都要全部回滚，否则数据不一致

**项目代码：**
```java
@Transactional  // 报告+偏差+支持原子操作
public Result<SysStageReport> submit(...) {
    // 1. 保存报告
    SysStageReport saved = reportService.submit(stageId, report, userId);
    // 2. 自动生成偏差
    if (isDeviation) deviationService.save(deviation);
    // 3. 自动生成支持
    if (needSupport) supportItemService.save(supportItem);
}
```

### 3.4 后端：Excel 导入导出（Apache POI）

**项目代码位置：**
- `OaProjectRowImporter.java` — 逐行导入 OA 项目
- `ProjectExportServiceImpl.java` — 导出项目台账
- `Result.java` — 导入结果封装

**面试会问：**
1. **POI 的 HSSF 和 XSSF 区别？**
   - HSSF：处理 `.xls`（Excel 97-2003），最大 65536 行
   - XSSF：处理 `.xlsx`（Excel 2007+），无行数限制
   - 本项目用 `WorkbookFactory.create()` 自动识别

2. **怎么处理大文件？**
   - SXSSF（流式写入）：只保持 100 行在内存
   - 用 `InputStream` 而非全量 `byte[]`

3. **导入怎么保证数据一致性？**
   - 每行一个事务（`REQUIRES_NEW`）
   - 失败回滚单行，不影响其他行
   - 返回 `OaProjectImportResult` 汇总成功/跳过/失败

### 3.5 前端：Vue 3 Composition API

**项目代码位置：**
- 全部 `views/*.vue` 文件
- `stores/auth.js` — Pinia 状态管理

**面试会问：**
1. **Composition API vs Options API？**
   - Composition：逻辑按功能聚合，更好的 TypeScript 支持
   - Options：data/methods/computed 分散，小型组件更直观

2. **`ref` vs `reactive`？**
   - `ref`：包装基本类型和对象，用 `.value` 访问
   - `reactive`：只包装对象，直接访问属性
   - 本项目：简单值用 ref，表单对象用 reactive

3. **`computed` vs `watch`？**
   - computed：依赖变化自动计算，有缓存
   - watch：监听变化执行副作用（如调 API）

4. **组件通信方式？**
   - Props down, Events up（父子）
   - Pinia store（跨组件）
   - Router params（页面间）

### 3.6 前端：Vue Router 权限控制

**项目代码位置：**
- `router/index.js` — 路由定义 + 守卫

**面试会问：**
1. **Hash 模式 vs History 模式？**
   - Hash：`#/projects`，兼容性好，不需要服务端配置
   - History：`/projects`，需要服务端 fallback

2. **路由守卫怎么实现权限？**
   ```js
   router.beforeEach((to, from) => {
     if (to.meta.roles && !to.meta.roles.includes(user.role)) {
       return '/dashboard'  // 无权限，踢回工作台
     }
   })
   ```

3. **动态路由 vs 静态路由？**
   - 本项目用静态路由 + meta.roles 白名单
   - 动态路由适合权限配置化的系统（后端返回菜单）

### 3.7 前端：Axios 拦截器

**项目代码位置：**
- `api/index.js` — 请求/响应拦截器

**面试会问：**
1. **请求拦截器做什么？**
   - 统一加 baseURL、timeout
   - 本项目用 Cookie 认证，不需要手动加 Token

2. **响应拦截器做什么？**
   - 统一错误处理（401→跳登录、403→提示无权限）
   - 统一数据提取（`return res.data`）

3. **怎么处理并发请求？**
   - `Promise.all()` — 本项目 Dashboard 加载统计使用
   - `axios.all()` 语法糖

### 3.8 数据库：表设计与索引

**项目代码位置：**
- `sql/init.sql` — 核心表
- `sql/v0.11.sql` — 索引建议

**面试会问：**
1. **为什么不用外键？**
   - 院内系统数据量小，应用层保证完整性即可
   - 外键会降低写入性能（每次 INSERT 要检查父表）

2. **索引设计原则？**
   - 高频查询列：`status` `project_id` `assignee_id`
   - 复合索引：`(project_id, status)` 最左前缀
   - 唯一索引：`contract_no` `username`

3. **怎么排查慢查询？**
   - `EXPLAIN SELECT ...` 看执行计划
   - `type=ALL` 全表扫描 → 需要加索引
   - `rows` 越大越慢

## 七、给面试官讲这个项目（3 分钟模板）

> "我做的是一个**林业规划院的项目全过程管理系统**，替代他们原来用的 Excel 台账。
>
> 技术栈是 Spring Boot 3 + Vue 3 + MyBatis-Plus + MySQL。后端 17 个 Controller、56 个 API，前端 23 个页面。
>
> 权限用了 Spring Security 的 Session 认证，四个角色——管理员、项目负责人、工程师、院领导——各有不同的菜单和接口权限。权限校验不仅在路由层做了，每个后端接口也都二次拦截。
>
> 核心业务流程是：OA 导入项目 → 负责人邀请工程师 → 分配阶段 → 工程师填报 → 负责人审阅 → 偏差/支持/变更处理 → 项目结项。结项前系统会逐项检查条件（阶段是否全完成、偏差是否全关闭等），不满足就列出原因。
>
> 技术上比较有意思的地方：OA 导入用了 Apache POI 解析 Excel，每行一个独立事务保证失败不污染其他行；填报提交加了 @Transactional 保证报告+偏差+支持三张表的原子写入；Excel 导出可以还原原始台账的格式。
>
> 如果重来我会加 Redis 缓存看板数据、用 WebSocket 做实时通知、补上集成测试和 CI/CD。"

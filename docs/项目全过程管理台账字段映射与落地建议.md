# 项目全过程管理台账字段映射与落地建议

> 来源：`项目全过程管理台账2.26(1).xlsx` 的“修改”工作表。目标是把领导给出的横向 Excel 台账，逐步沉淀成可填报、可审阅、可导入导出的业务系统。

## 1. 总体判断

当前系统已经覆盖了项目、成员、阶段、阶段填报、偏差、支持事项、变更、成果审批、复盘、经验库和导出等主要对象，但 Excel 的 37 列仍然更像“横向汇总台账”。如果继续把字段直接散落在 `sys_project`、`sys_stage_report`、`sys_review` 等固定表中，短期可演示，长期会遇到三个问题：

- Excel 导入导出很难无损还原原始表头层级。
- 不同项目类型需要不同字段时，需要频繁改表、改实体、改前端页面。
- 阶段填报历史和最终台账字段之间缺少明确映射，后期统计会不稳定。

建议下一版采用“核心业务表 + 阶段模板字段配置 + 字段值表”的方式：固定表保存项目、成员、阶段、流程状态；Excel 里的大部分栏目进入字段模板和字段值表。

## 3. 推荐数据模型 V1.1

### 3.1 保留固定业务表

- `sys_project`：项目基本信息、客户分析、前期分析、策划启动等稳定字段。
- `sys_project_member`：项目负责人、技术负责人、一线作业人员、经营责任人等成员角色。
- `sys_project_stage`：项目阶段主记录，保存计划时间、实际时间、责任人、状态。
- `sys_stage_report`：阶段填报主记录，保存通用填报字段、附件元数据、审阅状态。
- `sys_deviation`：偏差台账。
- `sys_support_item`：上级支持事项。
- `sys_change`：变更控制。
- `sys_approval`：成果评审与审批。
- `sys_review`：项目自评。
- `sys_experience`：经验总结。

### 3.2 新增配置型字段表

建议新增三张表，用来承接 Excel 的多级表头和阶段专属字段：

```sql
CREATE TABLE sys_stage_template (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    stage_code VARCHAR(50) NOT NULL,
    stage_name VARCHAR(100) NOT NULL,
    gate_name VARCHAR(100),
    sort_order INT NOT NULL DEFAULT 0,
    enabled TINYINT(1) DEFAULT 1
);

CREATE TABLE sys_stage_field_template (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    stage_code VARCHAR(50) NOT NULL,
    field_code VARCHAR(100) NOT NULL,
    field_label VARCHAR(200) NOT NULL,
    field_type VARCHAR(30) DEFAULT 'textarea',
    excel_column_index INT,
    excel_header_path VARCHAR(500),
    required TINYINT(1) DEFAULT 0,
    sort_order INT NOT NULL DEFAULT 0
);

CREATE TABLE sys_stage_field_value (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    report_id BIGINT NOT NULL,
    field_code VARCHAR(100) NOT NULL,
    field_value TEXT,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_report_field (report_id, field_code)
);
```

这样做后，外业调查、内业整理、成果提交可以共用同一个阶段填报页面，但字段由后端模板控制；Excel 导出也可以按 `excel_column_index` 和 `excel_header_path` 还原领导表格。

## 4. 下一版打磨优先级

### P0：不影响当前演示，先补业务准确性

1. 在阶段填报中增加“偏差原因”“是否需要上级支持”“支持事项标题”。
2. 阶段填报提交时，如果勾选偏差，自动生成 `sys_deviation`。
3. 如果勾选上级支持，自动生成 `sys_support_item`。
4. 负责人审阅通过后，把最新填报回写到 `sys_project_stage.actual_start/actual_end/status`。
5. 负责人退回后，工程师“我的待填报”重新显示该阶段。

### P1：把 Excel 做成真正可交付

1. 用 `sys_stage_template` 初始化“客户分析、项目前期分析、策划、启动、项目获取、外业调查、内业整理、成果提交、变更情况、成果评审、自评、经验总结”等阶段。
2. 用 `sys_stage_field_template` 保存 Excel 的 37 列表头路径。
3. 导出时按模板生成接近原 Excel 的横向台账。
4. 导入时先生成错误报告，再写入项目和阶段数据。

### P2：真实试运行需要补齐

1. 补部门、科室、项目类型、客户等级等字典。
2. 给项目成员增加“技术负责人”“经营责任人”等角色。
3. 增加审计日志：谁在什么时间改了什么字段。
4. 增加通知规则：逾期、未填报、待审阅超时、支持事项超时。
5. 增加试运行数据包：至少 3 个真实或模拟项目跑完整流程。

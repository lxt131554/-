## 三、发现的问题 🔴

### P0 — 必须修

| # | 问题 | 实测结果 | 根因 | 修复方向 |
|---|---|---|---|---|
| 1 | **拒绝邀请不可用** | engineer 点拒绝→`403 无权操作该数据` | `DELETE /...members/{mid}` 需要 manager 权限 | 新增 `PUT /.../members/{mid}/reject`，校验本人操作 |
| 2 | **登录失败 HTTP 返回 200** | 错误密码→`HTTP 200`(JSON code=401) | SecurityConfig 的 failureHandler 或 filter 未设 HTTP status | `LoginFailureHandler` 设置 `response.setStatus(401)` |
| 3 | **admin 可停用自己** | `PUT /api/users/1/toggle`→200 | UserController.toggle 无自保护检查 | 加判断：`if (id == 当前用户ID) return fail` |

### P1 — 应该修

| # | 问题 | 实测结果 | 修复方向 |
|---|---|---|---|
| 4 | **不存在资源返回 403 非 404** | `GET /api/projects/99999`→403 | 权限检查在资源查询之前，应先查资源→不存在返回404→存在再查权限 |
| 5 | **不存在附件返回 403 非 404** | `GET /api/reports/99999/attachment`→403 | 同上 |
| 6 | **超大 pageSize 无限制** | `size=10000`→200 正常返回 | 后端 limit 参数校验 |
| 7 | **leader 有 OA 导入权限** | leader 可调用 `POST /api/projects/import/oa` | `requireLeaderOrAdmin`→`requireAdmin` |
| 8 | **登录 HTTP 状态码** | 登录失败时 HTTP 200 但 JSON code=401 | 同 P0#2 |

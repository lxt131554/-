# 第二优先级问题修正设计

## 范围

- 保留院长查看收尾资料详情的现有实现。
- 不修改工程师“历史退回”统计口径，当前页面数据一致时不做无依据调整。
- 修正项目阶段完成后的提示、工程师工作台英文状态、管理员工作台摘要、成员移除原因留痕，以及偏差、支持、经验列表的权限分页。

## 业务规则

1. 进行中项目存在未完成阶段时，显示实际当前阶段。
2. 进行中项目的全部阶段均已完成时，显示“阶段已完成，待负责人确认项目完成”。
3. 已完成项目显示“项目已完成”；没有阶段的进行中项目显示“待负责人安排阶段”。
4. 工程师工作台不直接显示数据库英文状态。
5. 管理员工作台展示用户、项目、待审核、异常事项及 OA 导入摘要。OA 摘要使用现有项目数据计算累计导入数和最近导入时间，不新增导入历史表。
6. 移除成员由负责人或管理员确认后立即生效，不要求被移除成员确认。移除原因随请求发送，并写入一条状态为“已确认”的项目变更记录。
7. 偏差、支持和经验列表必须先限定当前用户可见项目，再执行数据库分页；管理员和院长保持全院范围。

## 验证

- 前端真实后端模式构建通过。
- 工程师工作台所有阶段状态均显示中文。
- 管理员工作台各卡片可跳转到对应页面。
- 成员移除后，项目变更记录能够看到成员、角色和移除原因。
- 普通用户列表分页的 `total` 与其可见数据一致，翻页不出现因二次过滤导致的空页。

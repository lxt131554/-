# 院内试运行交付说明

## 运行环境

- JDK：17
- Node.js：18 或 20
- MySQL：8.x，字符集建议 `utf8mb4`
- 后端端口：默认 `8080`
- 前端开发端口：默认 `5173`

## 后端配置

本地开发默认读取 `pm-backend/src/main/resources/application.yml`，可以直接使用本机 `pm_system` 数据库。

院内部署时不要改代码里的配置，使用环境变量覆盖：

```bash
SPRING_PROFILES_ACTIVE=prod
SERVER_PORT=8080
DB_URL=jdbc:mysql://数据库地址:3306/pm_system?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
DB_USERNAME=pm_user
DB_PASSWORD=实际密码
APP_CORS_ALLOWED_ORIGIN_PATTERNS=http://前端访问地址,https://前端访问地址
LOG_PATH=./logs
```

## 前端配置

本地开发默认：

```bash
VITE_API_BASE_URL=/api
VITE_MOCK=false
```

Netlify 演示环境如果继续使用纯前端 Mock，可以保留 `pm-frontend/.env.production` 中的：

```bash
VITE_MOCK=true
```

院内部署真实后端时，参考 `pm-frontend/.env.production.example`，设置：

```bash
VITE_API_BASE_URL=/api
VITE_MOCK=false
```

如果前后端不同域名，需要把 `VITE_API_BASE_URL` 改成真实后端地址，并同步配置后端 `APP_CORS_ALLOWED_ORIGIN_PATTERNS`。

## 启动检查

后端启动后先访问：

```text
GET /api/health
GET /actuator/health
```

`/api/health` 会返回服务状态、当前 profile 和数据库连通性。若 `database` 为 `DOWN`，优先检查数据库地址、账号、密码、库名和网络连通性。

## 日志定位

后端每个请求会生成 `X-Request-Id`，同时写入响应头和接口返回体。前端错误提示中也会附带 requestId。

后端日志位置默认：

```text
pm-backend/logs/pm-backend.log
```

排查问题时，用前端提示里的 requestId 到日志中搜索即可定位同一次请求。

## 数据库脚本

首次建库仍按现有脚本执行。第二轮新增的 `sql/v0.11.sql` 是试运行建议索引脚本，部署前确认表已存在后执行一次，不要重复执行。

## 版本注意

当前命令行若使用 Node 16，`npm run build` 会因为 Vite 6 缺少 `crypto.getRandomValues` 失败。请切换 Node 18/20 后再构建。

当前后端项目配置为 Java 17。若 VSCode Java 插件或命令行使用 Java 11，会出现编译/启动异常，需要切到 JDK 17。
